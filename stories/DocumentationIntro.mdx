# Summy 디자인 시스템

Summy 애플리케이션의 디자인 시스템에 오신 것을 환영합니다. 이 문서는 개발자와 디자이너가 Summy의 UI 컴포넌트를 이해하고 활용하는 데 도움이 됩니다.

## 주요 컴포넌트

Summy의 디자인 시스템은 다음과 같은 핵심 컴포넌트로 구성되어 있습니다:

### 폼 요소
- **Input**: 사용자 입력을 받기 위한 텍스트 필드
- **Select**: 옵션 목록에서 선택할 수 있는 드롭다운
- **Checkbox**: 토글 선택을 위한 체크박스
- **RadioGroup**: 상호 배타적인 옵션 선택을 위한 라디오 버튼 그룹
- **Textarea**: 여러 줄의 텍스트 입력을 위한 영역

### 액션 요소
- **Button**: 사용자 행동을 트리거하는 버튼
- **IconButton**: 아이콘만 있는 간결한 버튼

### 레이아웃 요소
- **FormControl**: 폼 요소의 레이아웃을 제어
- **Card**: 콘텐츠를 담는 카드 컴포넌트
- **Container**: 페이지 콘텐츠를 감싸는 컨테이너

## 디자인 시스템 원칙

1. **일관성**: 모든 UI 요소는 일관된 시각적 언어를 사용합니다.
2. **접근성**: 모든 컴포넌트는 WCAG 2.1 AA 표준을 준수합니다.
3. **재사용성**: 컴포넌트는 다양한 상황에서 재사용할 수 있도록 설계되었습니다.
4. **유연성**: 컴포넌트는 여러 변형과 상태를 지원합니다.

## 테마 시스템

Summy는 CSS 변수 기반의 테마 시스템을 사용합니다. 이를 통해 색상, 간격, 그림자 등의 디자인 토큰을 일관되게 관리할 수 있습니다.

```css
:root {
  --color-primary-500: #8b5cf6;
  --color-secondary-500: #14b8a6;
  --spacing-md: 1rem;
  --radius-lg: 0.5rem;
}
```

## 컴포넌트 사용법

각 컴포넌트의 사용법과 예제는 해당 컴포넌트의 문서 페이지에서 확인할 수 있습니다. 기본적인 사용 예시:

```jsx
import { Button } from '../components/ui/Button';

function MyComponent() {
  return (
    <Button variant="primary" size="md">
      버튼 텍스트
    </Button>
  );
}
```

## 컨트리뷰션

디자인 시스템에 기여하고 싶으신가요? 다음 가이드라인을 따라주세요:

1. 새로운 컴포넌트를 추가할 때는 기존 디자인 패턴을 준수해 주세요.
2. 모든 컴포넌트는 접근성을 고려하여 개발해야 합니다.
3. Storybook 문서화를 통해 사용 예시와 변형을 명확히 보여주세요.
4. 테스트 코드를 작성하여 컴포넌트의 품질을 유지해 주세요.
